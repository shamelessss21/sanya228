# Пояснения для курсовой работы

## 1) Что было исправлено

По вашему запросу практическая часть **возвращена на C++**.

Сделано:

1. Убрана Java-реализация из рабочего решения.
2. Добавлена новая C++-реализация с понятной архитектурой:
   - `Task` — описание задачи;
   - `Dag` — хранение графа зависимостей;
   - `Scheduler` — многопоточное исполнение с контролем зависимостей.
3. Реализована корректная обработка ошибок:
   - проверка существования задач;
   - проверка на циклы в графе;
   - проброс исключений из рабочих потоков.
4. Добавлена стандартизированная сборка через CMake.
5. Добавлены автоматические тесты (топологическая сортировка, циклы, исполнение задач, обработка исключений).

## 2) Что можно написать в защите

- Использован алгоритм Кана для топологической сортировки DAG.
- Многопоточный запуск построен так, что задача попадает в очередь только после выполнения всех ее предшественников.
- Планировщик завершает работу только после выполнения всех задач либо при критической ошибке.
- Наличие циклов определяется до старта выполнения, что исключает зависание планировщика.
- Корректность ключевых сценариев подтверждена автоматическими тестами.

## 3) Формулировка про улучшения качества

В итоговой версии повышена надежность решения за счет строгой валидации входного графа, потокобезопасного управления очередью готовых задач, детерминированного завершения пула потоков, автоматизированной проверки ключевых сценариев и явной обработки исключений во время выполнения задач.
